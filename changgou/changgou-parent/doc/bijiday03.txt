+ SKU 和 SPU (商品)

	+ SPU 标准的产品单位 (理解一个逻辑上商品, 不能出货的)    1 
	+ SKU 库存量单位(理解成多个规格组合而成的商品,能出货的)	 N

	parent_id` int(20) DEFAULT NULL COMMENT '上级ID',  如果parent_id=0 表示该分类为一级分类.


+ 商品分类的列表展示
	+ 前端根据点击到的父分类的ID 传递后台
	+ 后台接收之后,根据父ID 查询该父id下的子分类列表 
	+ 返回List<category>

+ 商品的规格的列表展示
	+ 根据点击到的分类的ID
	+ 查询分类对应的模板的ID
	+ 根据模板的ID 获取该模板对应的所有的规格的列表数据

+ 商品的参数列表的展示
	+ 根据点击到的分类的ID
	+ 查询分类对应的模板的ID
	+ 根据模板的ID 获取模板对应的参数表的列表数据()


+ 添加商品的功能
	+ 页面编写商品的数据
	+ 将其作为请求体 传递后台 
	+ 后台先创建POJO(组合对象:一个SPU 对应多个SKU)来接收请求体()
	+ 后台接收之后 获取SPU的数据和SKU的列表数据,将数据存到 spu 和sku表中
	+ 返回 前端一个成功()

+ 商品的审核  上架  下架

	+ 商品审核 修改一个字段的状态值. 要求:未被删除的商品  才能被审核

	+ 商品的上架  修改一个租单的状态值.要求:未被删除的商品 要被审核之后, 才能上架

	+ 商品的下架  修改一个租单的状态值.要求:未被删除的商品   下架完成之后,审核的状态变成(未审核) 


+ 商品的删除 (逻辑删除)
	+ 要求: 必须先下架 再删除

+ 物理删除
	+ 要求 :必须先逻辑删除 再物理删除

+ 商品的还原
	+ 修改状态
		还原之后,状态为 未审核.
